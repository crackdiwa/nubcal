# NubCal — Design Doc (2026-07-08)

## เป้าหมาย
แอพนับแคลภาษาไทยสำหรับใช้เองบนมือถือ: บันทึกอาหารได้ใน 10 วินาที เห็นทันทีว่าวันนี้เหลือกินได้อีกกี่แคล

## การตัดสินใจหลัก (คุยกับเจ้าของโปรเจกต์แล้ว)
- **AI วิเคราะห์รูป/ข้อความ: Gemini** (`gemini-2.5-flash`) — ผู้ใช้ขอ key ฟรีจาก aistudio.google.com ใส่ครั้งเดียว เก็บใน localStorage เรียกตรงจากเบราว์เซอร์
- **Login: ในเครื่อง (localStorage)** — หลายบัญชีต่อเครื่องได้ รหัสผ่านเก็บเป็น SHA-256(salt+pass) ไม่มีเซิร์ฟเวอร์
- **ไฟล์เดียว** (`index.html`) ตามสไตล์โปรเจกต์ anutung-electric · deploy ด้วย GitHub Pages

## สถาปัตยกรรม
Vanilla HTML/CSS/JS ไฟล์เดียว ไม่มี dependency นอกจาก Google Fonts (Prompt + Anuphan)

### ข้อมูลใน localStorage
| key | เก็บอะไร |
|---|---|
| `nc_users` | ผู้ใช้ทั้งหมด `{id, un, salt, hash, profile}` |
| `nc_session` | id ผู้ใช้ที่ล็อกอินอยู่ |
| `nc_log_<uid>` | บันทึกอาหาร `{ "YYYY-MM-DD": [entry...] }` |
| `nc_gemini_key` | Gemini API key (ต่อเครื่อง) |

entry = `{id, ts, meal(b/l/d/s), src(photo/scan/db/ai), name, portion, qty, k, p, c, f, na, s}` — ค่าต่อ 1 หน่วย คูณ qty ตอนแสดงผล

### การคำนวณเป้า (คำนวณในเครื่อง ไม่ใช้ AI)
- BMR (Mifflin–St Jeor) × activity (1.2–1.9) = TDEE
- ลดไขมัน −500 / รักษา 0 / เพิ่มกล้าม +300 / เพิ่มน้ำหนัก +500 (มี floor 1200/1500 แคล)
- โปรตีน 1.4–1.8 ก./กก.ตามเป้าหมาย · ไขมัน 27.5% พลังงาน · คาร์บส่วนที่เหลือ
- โซเดียม ≤ 2,000 มก. · น้ำตาล ≤ 10% พลังงาน (WHO)

### ช่องทางบันทึกอาหาร 3 ทาง → confirm sheet เดียวกัน (แก้จำนวน/มื้อได้) → save
1. **รูปถ่าย**: ย่อรูป ≤1024px → Gemini generateContent + JSON schema → รายการอาหาร
2. **บาร์โค้ด**: BarcodeDetector (กล้อง) หรือพิมพ์เลข → Open Food Facts v2 → ต่อหน่วยบริโภค/100g
3. **พิมพ์เอง**: ค้นฐานข้อมูลอาหารไทย ~110 เมนูในไฟล์ หรือส่งข้อความให้ Gemini ประเมิน

### Error handling
ทุก error เป็นภาษาไทยอ่านรู้เรื่อง: ไม่มี key → พาไปหน้าโปรไฟล์ · key ผิด (400/403) · โควต้าหมด (429) · เน็ตหลุด · บาร์โค้ดไม่เจอ → แนะนำทางเลือกถัดไปเสมอ

## หน้าจอ
auth → setup (ครั้งแรก/แก้ไข พร้อม live preview เป้า) → app shell 3 แท็บ (วันนี้ / ย้อนหลัง / โปรไฟล์) + FAB เพิ่มอาหาร

## ดีไซน์
โทน "ตลาดเช้า": เขียวกะเพรา `#1B4D33` + แดงพริก `#D94F3D` (ใช้เตือนเกินเป้า) · ฟอนต์ตัวเลข/หัวข้อ Prompt, เนื้อความ Anuphan · hero เป็น gauge 270° บอกแคลที่เหลือ · วันที่แสดงแบบ พ.ศ.

## เพิ่มเติม (2026-07-08 รอบสอง): บันทึกประจำวัน + คาดการณ์ถึงเป้า
- โปรไฟล์เพิ่ม `target` (น้ำหนักเป้าหมาย, ไม่บังคับ)
- `nc_diary_<uid>` = `{ "YYYY-MM-DD": {w, note} }` — การ์ด "บันทึกประจำวัน" ในหน้าแรก จดน้ำหนัก+โน้ตของวันนั้น จดน้ำหนักแล้วอัปเดต `profile.weight` เป็นค่าล่าสุดและคำนวณเป้าใหม่ทันที
- `predictGoal()`: มีบันทึกน้ำหนัก ≥2 จุด ห่าง ≥7 วัน → อัตราจริง (กก./สัปดาห์) ไม่งั้นใช้อัตราตามแผน (adj×7 ÷ 7700 แคล/กก.) — สถานะ: notarget / reached / maintain / stalled / wrongway / eta
- หน้าย้อนหลังเพิ่มกราฟเส้นน้ำหนัก (จุดเฉพาะวันที่จด, เส้นประเป้า) และลิสต์วันรวมวันที่มีแค่บันทึก (แสดง "—" แทนแคล) ส่วนค่าเฉลี่ย/วันที่อยู่ในเป้านับเฉพาะวันที่จดอาหาร

## เพิ่มเติม (2026-07-08 รอบสาม): ผู้ใช้ปรับค่าได้ทุกอย่าง
- confirm sheet: ทุกรายการ (จาก AI / บาร์โค้ด / เมนูในตัว) มีปุ่ม "แก้ไขค่าเอง" เปิดฟอร์มแก้ ชื่อ, ปริมาณ/หน่วย และสารอาหารครบ 6 ค่า (ค่าต่อ 1 หน่วย, ตัวคูณ qty ยังทำงานเหมือนเดิม) — อัปเดต state ตอน input, รีเฟรชตัวเลขสรุปตอน change
- sheet พิมพ์เอง: ปุ่ม "กรอกค่าเอง" เปิด confirm sheet พร้อมรายการเปล่าที่ฟอร์มเปิดอยู่ (ชื่อว่าง → default "อาหารของฉัน")
- แตะรายการที่บันทึกแล้วในหน้าแรก → `openEntryEditor()` เปิดตัวแก้ไข (S.editing = {dk, id}) แก้ค่า/ย้ายมื้อแล้วบันทึกทับ id เดิม

## เพิ่มเติม (2026-07-09): กันข้อมูลหาย + สแกนแม่นขึ้น + ฐานข้อมูลใหญ่ขึ้น
- **กันข้อมูลหาย**: `navigator.storage.persist()` ตอนล็อกอิน (best-effort) + การ์ด "สำรองข้อมูล" ในโปรไฟล์ — export ทุกบัญชี (users + logs + diaries, ไม่รวม API key) เป็นไฟล์ .json / import กลับแบบ merge (user ชนกันด้วย id หรือ username → ใช้ของไฟล์, บันทึกรวมรายวันโดยวันซ้ำใช้ของไฟล์)
- **สแกนแม่นขึ้น**: จำกัด format เหลือ ean_13/ean_8/upc_a, ต้องอ่านได้เลขเดิม 2 เฟรมติด + ผ่าน checksum GTIN (`validGTIN`) ถึงยอมรับ, ช่องพิมพ์เลขเองก็เช็ค checksum, ถ้า OFF ไม่มีหน่วยบริโภคใช้ขนาดทั้งซอง/ขวดจาก `quantity` แทนการตีเป็น 100 ก., ปุ่ม "ถ่ายรูปฉลากให้ AI อ่าน" ใน sheet สแกน + system prompt สอนให้อ่านฉลากโภชนาการโดยตรง
- **FOOD_DB ขยายเป็น 271 เมนู** + ค้นหาแบบหลายคำ (ทุกคำต้องอยู่ในชื่อ)
